# Memory

- 2026-07-02: When Codex reports plugin load warnings from cached plugin files, validate the exact reported path and any mirrored marketplace source copy. Hook config JSON currently accepts only a top-level `hooks` field; do not add descriptive metadata there. For skill frontmatter descriptions containing quoted examples, use YAML-safe quoting and run the packaged `quick_validate.py` against the skill folder.
- 2026-07-06: For React effects keyed by route params, guard every async state commit with an effect-local `active` flag or monotonic ticket. This includes independent parallel fetches and follow-up enrichment requests, otherwise an older route can overwrite the currently displayed event after navigation.
- AsyncAPI governance corrections: validate operation `messages` refs by accepting both generated channel-scoped refs and direct `#/components/messages/...` refs; diff enum sets recursively through array `items` so removed item enum values stay breaking.
- AsyncAPI exporter examples: convert object-map payloads (`JObject`, `IDictionary`, and `IReadOnlyDictionary`) before the generic `IEnumerable` fallback so fluent examples export as JSON objects instead of arrays of entries.
- AsyncAPI merge conflict pattern: preserve independent extension inputs rather than choosing one side. When exporter code moves to `NimBus.ServiceBus.AsyncApi`, keep `NimBus.Core.Events.AsyncApiFormat` as the canonical SDK/provider type and use distinct aliases (`CoreAsyncApiFormat`, `ServiceBusAsyncApiExporter`) inside `NimBus.CommandLine` to avoid obsolete bridge type shadowing. Verify with CommandLine tests plus `dotnet build src/NimBus.sln`.
- Review correction pattern: before classifying surprising behavior as a defect, cross-check the relevant ADRs and spec decision/negative sections. Intentional compatibility trade-offs (for example ADR-012's handoff `ErrorType` wrapper) should be reported as accepted design debt, not bugs, unless the requested scope explicitly reopens the decision.
- Cross-surface status contracts: derive API/service eligibility from the authoritative enum plus documented CLI rules, not from a UI option list. A UI may intentionally expose only a common subset; do not silently narrow the backend contract to match it.
- Cancellation/checkpoint correction: when a cancellation handler retries durable bookkeeping, define and enforce an idempotent provider contract that preserves the first committed timestamp, and test both zero-commit and partial-commit cancellation. For async cleanup APIs without cancellation-token support, initiate cleanup, bound the wait with the host token, and observe eventual completion so shutdown cannot hang or leak task failures.
- Regression-review pattern: metadata getters that lazily parse untrusted bodies must memoize both successful and failed attempts, return a safe missing value on parse failure, and keep normalization in one owning layer so lifecycle and dead-letter paths cannot throw. Cleanup guards must cover recovery as well as shutdown; best-effort cancellation bookkeeping needs a bounded wait, the actual failure in diagnostics, and an always-running provider conformance test. Centralize provider logging before exception scrubbing, expose only safe status/retry fields, and make scoped APIs fail closed when no scope factory exists.
- Orchestration documentation correction: strict workflow correlation, message identity, and lineage cannot use `PublisherClient.Schedule(IEvent, ...)`, which generates native metadata and exposes no overrides. Use the intended decorated `ISender` with a validated prebuilt `IMessage`, guard against `TryAddSingleton<ISender>` shadowing, and never conflate topology-assigned `EventId` with durable timeout/workflow identity. Distinguish direct broker cancellation from outbox logical cancellation, and keep ADR examples aligned with SQL/Cosmos atomicity limits.
- AgentFactory workspace correction: use the canonical lowercase slug `nimbus` for the NimBus repository (`C:\\Git\\NimBus`). Before bulk task creation, verify the slug against `/api/workspaces` or an explicit scoped task listing; do not infer the workspace from the product's display casing or the MCP server's pinned default.
- AgentFactory task ordering: use first-class dependency relationships (`dependent` waits for `prerequisite`) instead of relying only on dependency prose in task specs. A dependency is satisfied only when the prerequisite is `done`; encode required sequencing as edges, keep soft coordination in prose, and verify the resulting DAG plus unmet counts after bulk updates.
