01 · Shift

From coded glue to governed mappings

The PR introduces a declarative path for connecting two known event contracts without making the transformation itself another hand-written service.

Conventional path

Write, deploy, and operate a custom integration

The transformation logic lives in application code, so changing the mapping follows a software delivery cycle.

→
PR #66 path

Propose, review, and activate a declarative mapping

An agent drafts JSONata against registered schemas; a human controls activation; NimBus executes the approved version.

03 · Gate

The mapping lifecycle

The PR description centers human approval: agent output begins as a draft and becomes executable only after an operator transition.

Paused
