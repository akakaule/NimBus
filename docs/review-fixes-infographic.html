Process-tree safety

Kill the tree, then confirm exit

Cancellation attempts Kill(entireProcessTree: true), waits for exit on a bounded cleanup token, logs expected race/access failures, and then rethrows the original cancellation.

  • The normal path waits for the Azure CLI tree before the ephemeral parameter-file scope unwinds.
  • Win32Exception, mid-exit races, and unsupported tree-kill behavior do not replace cancellation.
  • Tests launch a real descendant process and prove both parent and child terminate.
