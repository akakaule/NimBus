NimBus · Feature Brief · Spec 023

AI Integration Mapper

When two systems disagree about message shape, an AI agent writes the translation once, a human approves it, and NimBus applies it to every message — deterministically, with no integration code written or deployed.

v1 · demo-grade · implemented 🧠 AI proposes · 👤 you approve 🔒 no LLM at runtime

Why it matters

BEFORE

Every system-to-system connection needs a developer to hand-write and deploy a C# translation handler — then maintain it. New connection = more code.

AFTER

An AI agent reads both contracts and proposes a mapping. A human approves it in the dashboard. NimBus then translates every message automatically — audited end-to-end, no code.

What it does

1Discoversource & target contracts from the registry
2Authora reusable JSONata transform — AI thinks once
3Approvehuman review of worked examples (Draft→Active)
4Applydeterministically, validated vs target — no LLM
5Recoverpark bad output; mark Stale on source drift

How it works

A new Marketing app emits leads in its own shape; the mapper translates them into the canonical ERP customer contract — joining the existing CRM→ERP demo. The AI authors once; the platform runs it on every message.
