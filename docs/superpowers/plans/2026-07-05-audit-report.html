High PERF src/nimbus.core/outbox/outboxdispatcher.cs:52

One failing outbox row halts dispatch of the entire outbox indefinitely (head-of-line blocking, no poison handling)

DispatchPendingAsync breaks out of the loop on the first DispatchOneAsync failure ('Stop dispatching on first failure to maintain ordering'). DispatchOneAsync returns false on ANY exception, including deterministic ones: JsonConvert.DeserializeObject of a malformed Payload (line 113) or a send to a deleted topic. GetPendingAsync always returns oldest-first (SqlServerOutbox.cs:175-178 'ORDER BY [CreatedAtUtc] ASC'), so every subsequent poll re-fetches the same poison head row, fails again, and dispatches nothing. Neither IOutbox nor OutboxMessage has an attempt counter or failed state, so messages for ALL sessions and endpoints behind the bad row are blocked forever.

FixScope the ordering guarantee to its actual unit: on failure, record the failed row's SessionId (or To for session-less rows) in a skip-set and continue dispatching rows belonging to other sessions instead of breaking the whole batch. Add a DispatchAttempts/LastError column to OutboxMessage with a max-attempts threshold that parks the row (marks it failed and emits a metric/log) so a poison row cannot stall the outbox permanently.
