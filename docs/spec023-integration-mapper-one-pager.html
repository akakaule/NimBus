NimBus · Feature Brief

AI Integration Mapper

When two systems disagree about message shape, an AI agent writes the translation once, a human approves it, and NimBus applies it to every message — deterministically, with no integration code written or deployed.

Why it matters

BEFORE

Every system-to-system connection needs a developer to hand-write and deploy a C# translation handler — then maintain it. New connection = more code.

AFTER

An AI agent reads both contracts and proposes a mapping. A human approves it in the dashboard. NimBus then translates every message automatically — audited end-to-end, no code.

What it does

  1. Discovers the source & target contracts from the schema registry.
  2. Authors a reusable declarative transform (JSONata) — the AI thinks once.
  3. Human approves (Draft → Active) after reviewing worked examples.
  4. Applies it deterministically on every message — validated against the target schema, no LLM at runtime.
  5. Parks bad output & detects drift — invalid results are recoverable; a changed source contract marks the mapping Stale.

How it works — think once, run forever

A new Marketing app emits leads in its own shape; the mapper translates them into the canonical ERP customer contract — joining the existing CRM→ERP demo.
