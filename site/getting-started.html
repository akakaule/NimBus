NimBus · getting started

From zero to a session-ordered event flow in eight steps: define an event, publish it, handle it, provision the topology, run the whole platform with Aspire, then watch — and deliberately break — a message in the management WebApp. To see messages move through the runtime, open the live demo; the full written guide is docs/getting-started.md.

[SessionKey] makes ordering an entity-level guarantee: every message with the same session ID is processed FIFO. When a handler fails, the session blocks and later messages defer — Resubmit in the WebApp replays them in original order.
You'll need: .NET 10 SDK  ·  an Azure Service Bus namespace  ·  an Azure Cosmos DB account  ·  Docker Desktop (for the Aspire dashboard)
