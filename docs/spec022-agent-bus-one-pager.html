NimBus · Feature Brief

AI Agents as Bus Participants

Let AI agents (e.g. Claude) join the event platform as first-class peers — discovering, defining, exchanging and settling messages — through a standard agent API (REST + OpenAPI) and an MCP server. No code written or deployed.
v1demo-grade
implemented

Why it matters

BEFORE

Only hand-written C# services could publish or consume events. An AI agent had no way in — there was no door built for it.

AFTER

An external AI agent can discover the platform, invent its own message type at runtime, subscribe, publish (validated), and settle its work — all visible in the same audit trail as coded services.

What an AI agent can do

  1. Discover the platform — endpoints & event types.
  2. Define a new event type at runtime from a JSON schema — no C#, no deploy.
  3. Subscribe & receive matching messages (pull-based).
  4. Publish messages — auto-validated against the schema it defined.
  5. Settle work (done / failed) — tracked end-to-end.

How it works — the live demo

An AI enrichment agent joins the existing CRM → ERP demo: a coded service hands off to an LLM, which hands back to coded services — one continuous, audited chain.
